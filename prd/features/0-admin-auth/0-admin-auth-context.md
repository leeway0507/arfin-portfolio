# Epic 0: Admin 진입 및 인증

> 파일: `0-admin-auth-context.md` (폴더 `0-admin-auth/` 내)

## Context

포토그래퍼(친구)는 포트폴리오 사이트에 반영할 사진을 바꾸고 싶을 때마다 "개발자에게 또 부탁해야 하나?", "내가 직접 순서만 바꿀 수는 없을까?", "지금 올려둔 사진 목록이 어떻게 되어 있지?" 같은 생각을 한다. 개발자를 거치지 않고 스스로 Admin에 들어가 사진을 관리하고 싶어 한다.

포토그래퍼는 매번 사진 순서 변경이나 추가·삭제를 위해 개발자에게 요청하는 것이 부담스럽다는 걸 알고 있다. 자신이 원할 때 직접 사이트에 반영된 사진을 바꾸고 싶지만, 개발자에게 부탁하는 걸 반복하는 건 피하고 싶어 한다. 그래서 본인만 쓸 수 있는 Admin에 안전하게 들어갈 수 있는 방법을 원한다.

## Goal

포토그래퍼가 Google 계정으로 로그인하여 Admin 영역에 접근할 수 있게 한다. 인증된 사용자만 사진 관리 기능을 사용할 수 있도록 진입점을 제공한다.

## Scenario Overview

**Purpose**: Google 인증을 통해 Admin에 로그인하고, 사진 관리 화면으로 진입한다.

**User Situation**:

- 포트폴리오 사진을 직접 관리하려면 개발자에게 매번 요청해야 하는 상황에서, 본인만 접근 가능한 Admin이 필요하다.
- 본인 계정으로만 로그인되어 Admin을 쓸 수 있기를 원하며, 다른 사람이 사진을 수정하는 것은 원하지 않는다.
- Cloudflare Pages 기반 Static 환경에서 동작해야 하며, 인증은 **Firebase Authentication**을 사용한다. Google 로그인·허용 계정 관리는 Firebase로 처리하며, 단일 사용자(친구 한 명) 수준이면 충분하다.
- 모바일에서도 로그인·접근이 가능해야 한다.

**Core User Actions**:

- Admin 진입 경로에서 로그인 필요 상태를 인지한다.
- Google 계정으로 로그인한다.
- 로그인 성공 시 사진 관리(Epic 1) 화면으로 이동한다.
- 로그아웃하여 Admin 접근을 종료한다.

**Transitions**:

- Google 로그인 완료 → Epic 1 (사진 관리) 또는 `1-photo-management`
- 로그아웃 → 로그인 화면(본 에픽)으로 복귀
