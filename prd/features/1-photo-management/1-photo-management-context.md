# Epic 1: 사진 관리

> 파일: `1-photo-management-context.md` (폴더 `1-photo-management/` 내)

## Context

포토그래퍼(친구)는 Admin에 로그인한 뒤 "지금 사이트에 어떤 사진이 올라가 있지?", "새로 찍은 사진을 추가하려면 어떻게 하지?", "이 순서를 바꾸고 싶은데?" 같은 질문을 해결하고 싶어 한다. 개발자 도움 없이 사진을 올리고, 지우고, 순서를 바꿔서 포트폴리오를 손쉽게 갱신하고 싶어 한다.

포토그래퍼는 매번 개발자에게 사진 순서 변경이나 업로드를 요청하는 것이 부담스럽다는 걸 인식하고 있다. 원할 때 직접 사진을 추가·삭제·재정렬하고 싶지만, 기술적인 절차(파일 변환, 스토리지 업데이트, 코드 수정)에 매번 의존하는 것은 피하고 싶어 한다. 그래서 한 화면에서 사진 목록을 보고, 업로드·삭제·순서 변경을 할 수 있기를 원한다.

## Goal

포토그래퍼가 현재 포트폴리오에 반영된 사진 목록을 확인하고, 새 사진을 업로드하며, 불필요한 사진을 삭제하고, 노출 순서를 변경할 수 있게 한다. 업로드 시 이미지 최적화가 적용되어 사이트 성능을 유지하고, 편의 기능과 모바일 사용성을 제공한다.

## Scenario Overview

**Purpose**: 사진 목록을 보고, 업로드·삭제·순서 변경을 통해 포트폴리오 사진을 직접 관리한다.

**User Situation**:
- 개발자를 거치지 않고 사진을 올리고, 삭제하고, 순서를 바꾸고 싶어 한다.
- 여러 장을 한 번에 처리하고, 직관적인 조작(예: 드래그로 순서 변경, 드롭으로 업로드)을 원하며, 반복적인 부탁은 피하고 싶어 한다.
- 포트폴리오 페이지 로딩을 위해 원본(20~30MB) 대신 최적화된 이미지(1MB 이하, webp 등)가 업로드되어야 한다는 제약을 만족해야 한다.
- 모바일에서도 사진 확인·업로드·순서 변경이 가능해야 한다.

**Core User Actions**:
- 현재 포트폴리오에 반영된 사진 목록을 확인한다.
- 새 사진을 추가(업로드)한다. (최적화 후 저장)
- 포트폴리오에서 제거할 사진을 선택하고 삭제한다.
- 사진의 노출 순서를 변경한다.
- 여러 사진을 한 번에 업로드하거나 삭제한다.
- 필요 시 사진별 캡션(파일명 기반) 정보를 인지하고, 캡션을 따로 수정한다.

**Transitions**:
- 로그아웃 또는 "로그아웃" 액션 → Epic 0 (Admin 진입 및 인증) 또는 `0-admin-auth`
