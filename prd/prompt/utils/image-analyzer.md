## Overview

- 첨부된 이미지를 분석하여 개발자가 동일한 UI를 구현할 수 있는 상세한 디자인 스펙을 추출한다.
- 시각적 요소를 코드로 변환 가능한 구체적인 명세로 문서화한다.

## Input

이미지

- **이미지 파일**: 분석 대상 UI 스크린샷 또는 디자인 이미지
- **이미지 형식**: PNG, JPG, WebP 등 일반적인 이미지 포맷
- **이미지 내용**: 웹 UI 컴포넌트, 테이블, 폼, 대시보드 등

디자인 시스템 `vision/prompt/context/design-system.md`

## Output

다음 5개 섹션으로 구성된 구조화된 디자인 스펙 문서를 `{image_name}-design-spec.md` 파일로 생성한다.

1. **Overall Style**

   - 전체적인 디자인 스타일 특성 (예: minimal, high-density data table)
   - 디자인 시스템의 전반적인 특징

2. **Color Palette**

   - Backgrounds: 메인 배경, 헤더 배경, 행 배경 색상
   - Text Colors: 주요 텍스트, 보조/뮤트 텍스트 색상
   - Accents: 테두리, 호버 상태 색상

3. **Typography & Spacing**

   - 폰트 웨이트 사용: 헤더 vs 데이터 텍스트
   - 행 높이 및 셀 패딩: 데이터 밀도 정보

4. **Component Breakdown**

   - Badges/Tags: 정확한 모양 (rounded-md 등), 색상 로직 (파스텔 배경 + 어두운 텍스트 등)
   - Icons: 위치, 크기, 색상
   - Inputs/Controls: 체크박스 스타일, 필터 버튼 스타일

5. **Special Layout Features**
   - 고정 컬럼/헤더 여부
   - 계산된 필드 행 존재 여부
   - 기타 특수 레이아웃 기능

### 출력 품질 기준

- 모든 색상은 Tailwind CSS 클래스로 표현 가능한 형태로 명시
- 컴포넌트 크기와 간격은 픽셀 단위 또는 Tailwind spacing scale로 표기
- 모호한 표현 금지, 측정 가능한 수치 제시

## Step

**중요: 각 단계는 반드시 순차적으로 완료해야 한다. 한 번에 여러 단계를 진행하지 않는다. 각 단계 완료 후 중간 결과를 확인하고 다음 단계로 진행한다.**

### 1단계: 이미지 분석

**실행 규칙:**
- 이 단계만 먼저 완료한다. 다른 단계로 진행하지 않는다.
- 완료 후 사용자에게 보고하고 다음 단계로 진행한다.

**작업 내용:**
- 이미지 전체 구조 파악: 레이아웃 타입, 주요 컴포넌트 식별
- 시각적 계층 구조 분석: 정보 구조와 시각적 중요도 파악
- 디자인 시스템 패턴 추출: 반복되는 스타일 패턴 식별
- 분석 결과를 정리하여 사용자에게 보고

**완료 조건:**
- 이미지 구조 분석 완료
- 주요 컴포넌트 식별 완료
- 사용자에게 1단계 완료 보고

### 2단계: 스타일 요소 추출

**실행 규칙:**
- 1단계가 완료된 후에만 시작한다.
- 완료 후 사용자에게 보고하고 다음 단계로 진행한다.

**작업 내용:**
- 색상 팔레트 분석: 배경, 텍스트, 액센트 색상 추출 및 Tailwind 클래스 매핑
- 타이포그래피 분석: 폰트 크기, 웨이트, 행간 측정
- 간격 분석: 패딩, 마진, 컴포넌트 간 간격 측정
- 추출한 스타일 요소를 정리하여 사용자에게 보고

**완료 조건:**
- 색상 팔레트 추출 완료
- 타이포그래피 분석 완료
- 간격 분석 완료
- 사용자에게 2단계 완료 보고

### 3단계: 컴포넌트 상세 분석

**실행 규칙:**
- 2단계가 완료된 후에만 시작한다.
- 완료 후 사용자에게 보고하고 다음 단계로 진행한다.

**작업 내용:**
- UI 컴포넌트 식별: 버튼, 입력 필드, 배지, 아이콘 등
- 각 컴포넌트의 스타일 속성 정리: 크기, 색상, 모양, 상태별 스타일
- 상호작용 요소 분석: 호버, 포커스, 활성 상태 스타일
- 컴포넌트 분석 결과를 정리하여 사용자에게 보고

**완료 조건:**
- 모든 컴포넌트 식별 완료
- 컴포넌트 스타일 속성 정리 완료
- 사용자에게 3단계 완료 보고

### 4단계: 레이아웃 특성 분석

**실행 규칙:**
- 3단계가 완료된 후에만 시작한다.
- 완료 후 사용자에게 보고하고 다음 단계로 진행한다.

**작업 내용:**
- 레이아웃 구조 파악: 그리드, 플렉스, 테이블 등
- 고정 요소 확인: sticky 헤더, 고정 컬럼 존재 여부
- 반응형 특성 파악: 브레이크포인트별 변화 여부
- 레이아웃 분석 결과를 정리하여 사용자에게 보고

**완료 조건:**
- 레이아웃 구조 파악 완료
- 고정 요소 확인 완료
- 반응형 특성 파악 완료
- 사용자에게 4단계 완료 보고

### 5단계: 스펙 문서 작성

**실행 규칙:**
- 4단계가 완료된 후에만 시작한다.
- 5개 섹션을 순서대로 작성한다.
- 각 섹션 작성 후 다음 섹션으로 진행한다.

**작업 내용:**
- 표준 출력 형식에 따라 5개 섹션 순서대로 작성
  - Overall Style 섹션 작성
  - Color Palette 섹션 작성
  - Typography & Spacing 섹션 작성
  - Component Breakdown 섹션 작성
  - Special Layout Features 섹션 작성
- 기술적 정확성 검증: 모든 값이 코드로 구현 가능한 형태인지 확인
- 모호한 표현 제거: "적절히", "대략" 등의 표현을 구체적 수치로 대체
- 사용자에게 5단계 완료 및 문서 작성 완료를 보고

**완료 조건:**
- 5개 섹션 모두 작성 완료
- 기술적 정확성 검증 완료
- 모호한 표현 제거 완료
- 사용자에게 5단계 완료 보고

### 6단계: 검증 및 완성

**실행 규칙:**
- 5단계가 완료된 후에만 시작한다.
- 최종 검증만 수행한다.

**작업 내용:**
- 개발자 관점에서 구현 가능성 검토
- 누락된 스타일 요소 확인 및 보완
- 최종 스펙 문서 완성
- `{image_name}-design-spec.md` 파일로 저장
- 사용자에게 작업 완료를 보고

**완료 조건:**
- 구현 가능성 검토 완료
- 누락 요소 보완 완료
- 스펙 문서 파일 저장 완료
- 사용자에게 작업 완료 보고

## 실행 원칙

### 단계별 진행 규칙

1. **순차 실행**: 각 단계는 반드시 이전 단계가 완료된 후에만 시작한다.
2. **중간 확인**: 각 단계 완료 후 결과를 확인하고 사용자에게 보고한다.
3. **섹션별 작성**: 5단계에서는 각 섹션을 순서대로 작성한다.
4. **검증 필수**: 각 단계 완료 후 다음 단계로 진행하기 전에 완료 조건을 확인한다.
5. **구체적 수치**: 모호한 표현을 사용하지 않고 측정 가능한 수치를 제시한다.

### 금지 사항

- 한 번에 여러 단계를 동시에 진행하지 않는다.
- 단계 완료 전에 다음 단계로 진행하지 않는다.
- 중간 보고 없이 작업을 진행하지 않는다.
- 모호한 표현("적절히", "대략" 등)을 사용하지 않는다.
- 모든 섹션을 한 번에 작성하지 않는다.
