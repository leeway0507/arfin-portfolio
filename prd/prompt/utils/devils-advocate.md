## Overview

- 제공된 텍스트를 악마의 대변인(Devil's Advocate) 관점에서 비판적으로 검토한다.
- 논리적 허점, 가정의 문제점, 실행 가능성의 한계를 식별한다.
- 개선 방향과 대안을 제시하여 텍스트의 견고성을 높인다.

## Input

- **검토 대상 텍스트**: 프롬프트, 계획, 아이디어, 문서 등 검토가 필요한 모든 텍스트

## Output

- **검토 결과**: 비판적 분석과 개선 제안 (별도 파일로 저장하지 않음)
  - 발견된 문제점 목록
  - 각 문제점에 대한 구체적 지적
  - 개선 방안 제시

## Step

### 실행 프로세스

#### 1단계: 텍스트 구조 분석

- 텍스트의 목적과 핵심 주장 파악
- 논리적 구조와 전제 조건 식별
- 명시적/암묵적 가정 추출

#### 2단계: 비판적 검토

- **논리적 일관성 검증**
  - 논리적 모순이나 비약 지적
  - 인과관계의 타당성 검토
  - 예외 상황 고려 여부 확인
- **가정의 타당성 검증**
  - 암묵적 가정의 현실성 평가
  - 전제 조건의 타당성 검토
  - 맥락 의존성 문제 식별
- **실행 가능성 검증**
  - 구체성과 명확성 부족 지적
  - 리소스나 제약 조건 고려 여부 확인
  - 실현 가능성의 현실성 평가
- **완전성 검증**
  - 누락된 중요한 요소 식별
  - 대안 시나리오 고려 여부 확인
  - 위험 요소나 부작용 검토

#### 3단계: 개선 제안 작성

- 실행 가능한 개선 제안 중심으로 구성
- 문제점을 우선순위별로 정리

- 발견된 각 문제점에 대해 구체적 개선 방안 제시

### 작성 원칙

#### 톤앤매너

- 평어체 사용 (존댓말 금지)
- 객관적이고 건설적인 비판
- 감정적 공격 배제
- 문제 지적과 해결 제안의 균형

#### 검토 기준

- 비판은 구체적 근거와 함께 제시
- 모호한 지적 금지 ("좀 더", "나쁘다" 등)
- 개선 가능한 방향 명시
- 실행 가능한 대안 제시
